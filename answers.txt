 
1.What does git add do, and why is it necessary before committing?
It adds a change in the working directory to the staging area. It tells Git 
that you want to include updates to a particular file in the next commit

2.What does git commit do, and why is it necessary before pushing?
Commits help craft the history of your repository and include a short message 
explaining what was changed. 

3.What is the difference between git clone and git pull, and when should each be used?
Git clone creates a local copy of a remote repository, whereas git pull updates your
existing local repository with changes from a remote repository

4.How can you verify that a git pull was successful after making changes on GitHub?
By using the git log function
